Basemaps in StoryMapJS

Behind every great storymap is a basemap—literally!

The "basemap" for all webmaps, including storymaps, is the layer which shows land and water, roads and placenames. When we launched StoryMapJS, we included a few different basemap options created by Stamen Design and offered by them at no cost to you.

However, it turns out that the cost to Stamen Design was considerable, and in 2023, they recognized that they could no longer afford to provide this service. They arranged with a company called Stadia to take over hosting the maps.

Unfortunately, this change is going to impact most storymaps which have ever been created. Stadia offers a limited free level of service for non-commercial use, but commercial users and non-commercial maps after a certain point need to pay a fee.

Note that in the case of Stamen's watercolor basemap, a no-cost option is available. In 2021 Cooper Hewitt, Smithsonian Design Museum acquired Watercolor Maptiles into its digital design collection, and they make the tiles available for use in digital maps. You can now choose Cooper Hewitt: Watercolor from the "map type" menu in the StoryMap options configuration.

Effective immediately, new storymaps will default to using the OpenStreetMap: Standard basemap. We won't change existing storymaps yet, but eventually, for maps that aren't updated, the Stamen basemaps will stop working. At some point, we'll change StoryMapJS to switch those to using OpenStreetMap instead, but we want people to have time to learn about this change and consider their options. We will likely change maps using the Stamen Watercolor basemap to use the Cooper Hewitt Watercolor basemap, instead of changing them to use OpenStreetMap. You can change your storymap yourself, at any time.

If you want to continue to use a Stamen basemap on your storymap, it will need to be edited. Even if you qualify for the free level of service, you need to create an account, generate an API key so that Stadia can track usage, and update your storymap to configure that key. Stadia provides other basemaps besides the Stamen Design options. See below for more specific details on using Stadia maps with Stamen or other designs.

We also continue to support maps made with Mapbox Studio, another commercial service which has a no-cost level.

Basemap options, in detail

Changing the basemap of your storymap happens in the Options panel. Open the panel by clicking on the button in the top left of the editor window. Basemaps are chosen using the Map Type menu.

OpenStreetMap: Standard

OpenStreetMap is often described as the Wikipedia of mapping. Their basemap layer is the default for storymaps. It is available at no cost, although the organization accepts donations and offers memberships.

Cooper Hewitt: Watercolor

The Cooper Hewitt: Watercolor basemap is a no-cost option provided by Cooper Hewitt, Smithsonian Design Museum.

Mapbox

Mapbox is a commercial map service provider. If you create an account with them, you can use their existing styles or create your own using Mapbox Studio. When you configure a storymap to use a Mapbox basemap, you will enter the "style URL" (which looks something like mapbox://styles/nuknightlab/cikplzs9w00ddsykp7vqdz22u) and an access token created from your Mapbox account homepage.

Mapbox charges for their service based on monthly usage, but they offer a "free tier" if your total usage is under a certain amount.

Stadia

Stadia is a commercial map service provider. They offer a variety of basemap styles, including, as mentioned above, the styles originally designed by Stamen Design. To use Stadia maps, create an account. Once you've verified your email address, create a "property" in the Stadia dashboard. After doing that, in the "Authentication Configuration" section, click on "+ Add API Key." We understand that the recommendation on that page is to not use API keys in applications accessed through a web browser. However, the other option, based on domains, can't be used for storymaps made with our authoring tool, since all authors publish on the same domain. If you are a more technical user hosting storymaps on your own server, you could use the domain authorization control.

Once you have your API key, select the style you want to use from their themes page and click on the "Get started with theme name" button. Under the Preview and use [theme name] section, click on Standard URLs (or EU URLs, if you prefer), and copy the URL under the Raster XYZ PNG URL format header. It will look something like https://tiles.stadiamaps.com/tiles/alidade_smooth_dark/{z}/{x}/{y}{r}.png. In the StoryMapJS editor options pane, when you choose "Stadia", you will see places to enter the URL and API key.

Stadia charges for their service based on monthly usage, but have a "free tier" for non-profit users if your total usage is under a certain amount.

Gigapixel

Gigapixel is Knight Lab's term for StoryMaps made based on images instead of map data. Creating this kind of map requires some technical capability. Learn more on our Gigapixel page.

custom

This option will not be used by many storymap creators, but if you happen to have access to a basemap provider not mentioned above, you can choose this option and enter a map URL template. This template will typically look like a normal URL, except it will have, at least, {x}, {y} and {z} placeholders that StoryMapJS replaces with coordinates and a zoom level. It may also have {s} (for subdomains) and/or {r} (for Retina display support). Since this option will be so rarely used, we won't try to explain it in detail here, but you can always file a support request with us and we'll try to help.

Stamen: *

As discussed at length above, Stamen maps are no longer a viable option, and should not be chosen for new maps. Eventually, we will remove the options from the authoring tool.
